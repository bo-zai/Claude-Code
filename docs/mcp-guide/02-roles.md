# MCP 角色关系

## 四大核心角色

```
                    ┌─────────────────────────────────┐
                    │            用户                  │
                    │  (提问、审批、看结果)            │
                    └───────────────┬─────────────────┘
                                    │ ↑
                                    │ │ 用户输入 / 最终答案
                                    ↓ │
                    ┌─────────────────────────────────┐
                    │           Agent                  │
                    │  (协调者，软件程序)              │
                    └───────────────┬─────────────────┘
                                    │ ↑
                    Prompt + Tools  │ │ Tool Result
                                    ↓ │
                    ┌─────────────────────────────────┐
                    │           模型                  │
                    │  (AI 大模型，推理核心)          │
                    └───────────────┬─────────────────┘
                                    │ ↑
                    Tool Call       │ │ Tool Response
                                    ↓ │
                    ┌─────────────────────────────────┐
                    │         MCP Server              │
                    │  (工具提供者，外部程序)          │
                    └───────────────┬─────────────────┘
                                    │
                                    ↓
                    ┌─────────────────────────────────┐
                    │         外部资源                 │
                    │  (文件、数据库、API 等)          │
                    └─────────────────────────────────┘
```

---

## 角色 1：用户

### 定义
发起请求的人，整个系统的服务对象。

### 职责

```
┌─────────┐
│  用户   │
├─────────┤
│ 输入: 提出问题或任务                            │
│ 审批: MCP 工具调用时需要用户确认                   │
│ 输出: 看到最终答案                                 │
└─────────┘
```

### 交互方式

| 场景 | 用户行为 |
|------|----------|
| 提问 | 输入自然语言问题 |
| 工具审批 | 选择"允许本次" / "总是允许" / "拒绝" |
| 查看结果 | 阅读 AI 生成的答案 |

---

## 角色 2：Agent（关键角色）

### 定义
协调各组件的软件程序，是 AI 的"管家/运行环境"。

### 核心职责

```
┌─────────────────────────────────────────────────────┐
│                    Agent                            │
├─────────────────────────────────────────────────────┤
│                                                     │
│  1. 接收用户输入                                     │
│  2. 构建 Prompt 发给模型                             │
│  3. 解析模型的响应                                   │
│     ├─ 如果模型要调用工具 → 执行 MCP 调用           │
│     ├─ 如果模型要输出文本 → 展示给用户              │
│  4. 把工具结果反馈给模型                             │
│  5. 循环直到任务完成                                 │
│                                                     │
└─────────────────────────────────────────────────────┘
```

### 详细职责

| 阶段 | Agent 的工作 |
|------|-------------|
| 输入处理 | 解析用户消息、加载上下文、读取 MCP 工具列表 |
| Prompt 构建 | 将用户问题 + 工具定义 + 上下文打包成完整 Prompt |
| 输出解析 | 判断模型输出是文本还是工具调用 |
| MCP 通信 | 发送 JSON-RPC 请求、接收响应 |
| 权限管理 | 检查是否需要用户审批、处理审批结果 |
| 循环控制 | 决定是否继续调用工具还是结束对话 |

### 重要概念

**Agent 不是 AI，它是运行 AI 的软件程序！**

类比理解：
| 概念 | 类比 |
|------|------|
| 模型 | 大脑（思考） |
| Agent | 身体（执行大脑的指令） |
| MCP 工具 | 手脚（具体操作） |

---

## 角色 3：模型

### 定义
AI 大模型本身，是推理和决策的核心。

### 职责

```
┌─────────────────────────────────────────────────────┐
│                    模型                             │
├─────────────────────────────────────────────────────┤
│                                                     │
│  输入: Prompt（包含用户问题 + 工具列表 + 上下文）    │
│                                                     │
│  推理过程:                                          │
│  1. 分析用户想做什么                                 │
│  2. 判断是否需要调用工具                             │
│  3. 如果需要 → 输出工具调用请求                      │
│  4. 如果不需要 → 直接回答                            │
│                                                     │
│  输出:                                              │
│  ├─ 文本回答                                        │
│  └─ 工具调用请求（JSON格式）                         │
│                                                     │
└─────────────────────────────────────────────────────┘
```

### 工具调用决策

模型如何知道有哪些工具可用？Agent 会在每次请求的 Prompt 中包含工具定义：

```json
{
  "tools": [
    {
      "name": "query",
      "description": "查询代码知识图谱",
      "input_schema": {
        "type": "object",
        "properties": {
          "query": { "type": "string", "description": "查询内容" },
          "repo": { "type": "string", "description": "仓库名" }
        },
        "required": ["query"]
      }
    },
    {
      "name": "read_file",
      "description": "读取文件内容",
      "input_schema": {
        "type": "object",
        "properties": {
          "path": { "type": "string", "description": "文件路径" }
        },
        "required": ["path"]
      }
    }
  ]
}
```

模型根据这些定义来决定：
- 调用哪个工具
- 传递什么参数

## 角色 4：MCP Server

### 定义
提供工具和数据的接口，执行具体操作的外部程序。

### 职责

```
┌─────────────────────────────────────────────────────┐
│                  MCP Server                         │
├─────────────────────────────────────────────────────┤
│                                                     │
│  启动时:                                            │
│  1. 声明自己有哪些工具 (tools/list)                  │
│  2. 声明自己有哪些资源 (resources/list)             │
│                                                     │
│  运行时:                                            │
│  ├─ 接收 tools/call → 执行操作 → 返回结果           │
│  ├─ 接收 resources/read → 读取数据 → 返回内容        │
│                                                     │
│  能做什么:                                          │
│  ├─ 读取文件                                        │
│  ├─ 查询数据库                                      │
│  ├─ 调用 API                                        │
│  ├─ 操作代码仓库                                    │
│  └─ 任何你授权的操作                                │
│                                                     │
└─────────────────────────────────────────────────────┘
```

### 提供的三种能力

| 能力 | 说明 | 类比 |
|------|------|------|
| **Tools** | AI 可调用的函数，执行操作 | POST 请求 |
| **Resources** | AI 可读取的数据源 | GET 请求 |
| **Prompts** | 预定义的提示模板 | 快捷方式 |

### 与 Agent 的通信

MCP Server 通过标准协议（JSON-RPC）与 Agent 通信：

```
Agent                                    MCP Server
  │                                           │
  │──── tools/list ──────────────────────────→│ (你有哪些工具？)
  │←─── ListToolsResult ─────────────────────│ (这是我的工具列表)
  │                                           │
  │──── tools/call ──────────────────────────→│ (调用 query 工具)
  │←─── ToolResult ──────────────────────────│ (返回查询结果)
  │                                           │
```

## 角色间的数据流

```
用户输入
   │
   ↓
┌─────────────────────────────────────────────────────┐
│ Agent 接收输入                                       │
│   ├─ 加载上下文                                     │
│   ├─ 获取 MCP 工具列表                              │
│   └─ 构建 Prompt                                    │
└─────────────────────┬───────────────────────────────┘
                      │
                      ↓
┌─────────────────────────────────────────────────────┐
│ 模型处理                                            │
│   ├─ 分析问题                                       │
│   ├─ 决策：直接回答 or 调用工具                      │
│   └─ 输出结果                                        │
└─────────────────────┬───────────────────────────────┘
                      │
          ┌───────────┴───────────┐
          │                       │
          ↓                       ↓
    直接回答                  工具调用请求
          │                       │
          │                       ↓
          │           ┌───────────────────────┐
          │           │ Agent 执行 MCP 调用    │
          │           │   ├─ 权限检查          │
          │           │   ├─ 用户审批（如需要）│
          │           │   └─ 发送请求          │
          │           └───────────┬───────────┘
          │                       │
          │                       ↓
          │           ┌───────────────────────┐
          │           │ MCP Server 执行        │
          │           │   ├─ 执行操作          │
          │           │   └─ 返回结果          │
          │           └───────────┬───────────┘
          │                       │
          │                       ↓
          │           ┌───────────────────────┐
          │           │ Agent 反馈给模型       │
          │           │   └─ 构建新的 Prompt   │
          │           └───────────┬───────────┘
          │                       │
          │                       ↓
          │           ┌───────────────────────┐
          │           │ 模型再次处理          │
          │           │   └─ 基于工具结果回答  │
          │           └───────────┬───────────┘
          │                       │
          └───────────┬───────────┘
                      │
                      ↓
              展示给用户
```
