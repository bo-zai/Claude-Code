以下展示用户提问"帮我查询这个代码仓库的结构"的完整交互过程。


## 多轮工具调用

实际场景中，模型可能需要多次调用工具才能完成任务。

### 示例：分析代码调用关系

```
用户: "分析 src/auth.js 中的登录逻辑，找出调用它的地方"

时间线 ────────────────────────────────────────────────────→

第1轮:
  模型 → 调用 read 工具读取 src/auth.js
  MCP → 返回文件内容
  模型 → 分析代码，发现 login 函数

第2轮:
  模型 → 调用 context 工具查询 login 函数的调用者
  MCP → 返回调用者列表
  模型 → 综合分析，生成回答

最终回答: "login 函数被以下模块调用：
           - src/api/routes.js: handleLogin()
           - src/middleware/auth.js: authenticate()
           ..."
```

### Agent 循环逻辑

```python
# Agent 的核心循环伪代码
while True:
    # 1. 发送当前上下文给模型
    response = model.generate(context)

    # 2. 检查模型是否要调用工具
    if response.has_tool_calls():
        # 3. 执行所有工具调用
        for tool_call in response.tool_calls:
            # 检查权限
            if needs_permission(tool_call):
                if not user_approves(tool_call):
                    continue  # 跳过这个工具调用

            # 执行 MCP 调用
            result = mcp_server.call(tool_call)

            # 把结果加入上下文
            context.add_tool_result(result)

        # 继续循环，让模型处理工具结果
        continue

    # 4. 没有工具调用，展示最终答案
    display_to_user(response.text)
    break
```

## 权限控制机制

### 权限检查流程

```
┌─────────────────────────────────────────────────────┐
│                   权限检查流程                       │
├─────────────────────────────────────────────────────┤
│                                                     │
│  Agent 收到模型的工具调用请求                        │
│     ↓                                               │
│  查询权限配置                                        │
│     ├─ "allow" → 直接执行                            │
│     ├─ "ask" → 弹窗请用户审批                        │
│     └─ "deny" → 拒绝并告诉模型                       │
│     ↓                                               │
│  用户审批界面:                                       │
│  ┌─────────────────────────────────────────────┐   │
│  │ MCP 工具调用请求                             │   │
│  │                                              │   │
│  │ 工具: query                                  │   │
│  │ 参数: { query: "代码仓库结构" }              │   │
│  │                                              │   │
│  │ [允许本次] [总是允许] [拒绝]                 │   │
│  └─────────────────────────────────────────────┘   │
│     ↓                                               │
│  根据用户选择执行或拒绝                              │
│                                                     │
└─────────────────────────────────────────────────────┘
```

### 权限配置示例

```json
{
  "permissions": {
    "allow": [
      "read_file",        // 允许读取文件
      "query"             // 允许查询
    ],
    "ask": [
      "write_file",       // 写文件需要确认
      "execute_command"   // 执行命令需要确认
    ],
    "deny": [
      "delete_file"       // 禁止删除文件
    ]
  }
}
```

### 用户审批选项

| 选项 | 效果 |
|------|------|
| 允许本次 | 本次调用允许，下次还需要审批 |
| 总是允许 | 将此工具加入白名单，后续自动执行 |
| 拒绝 | 本次调用被拒绝，模型会收到错误信息 |

## 初始化握手流程

MCP Server 启动时，需要与 Agent 进行握手：

```
Agent                                    MCP Server
  │                                           │
  │──── initialize ──────────────────────────→│ (你好，我支持这些功能)
  │←─── InitializeResult ────────────────────│ (好的，我支持这些功能)
  │                                           │
  │──── initialized ─────────────────────────→│ (初始化完成，可以开始了)
  │                                           │
  │──── tools/list ──────────────────────────→│ (你有哪些工具？)
  │←─── ListToolsResult ─────────────────────│ (这是我的工具列表...)
  │                                           │
  │──── resources/list ───────────────────────→│ (你有哪些资源？)
  │←─── ListResourcesResult ──────────────────│ (这是我的资源列表...)
  │                                           │
  │         ... 正常交互阶段 ...               │
  │                                           │
```

## 错误处理

### 工具调用失败

```
Agent                                    MCP Server
  │                                           │
  │──── tools/call ──────────────────────────→│
  │←─── error response ───────────────────────│
  │                                           │
  │  {                                        │
  │    "jsonrpc": "2.0",                      │
  │    "id": 1,                               │
  │    "error": {                             │
  │      "code": -32602,                      │
  │      "message": "Invalid params:          │
  │                   missing required 'city'" │
  │    }                                      │
  │  }                                        │
  │                                           │
  ↓                                           │
Agent 把错误信息反馈给模型                      │
模型可以:                                       │
  - 重试（修正参数）                            │
  - 换一个工具                                  │
  - 告诉用户无法完成                             │
```

### 用户拒绝

```
用户点击"拒绝"
     ↓
Agent 告诉模型: "工具调用被用户拒绝"
     ↓
模型可以选择:
  - 解释为什么需要这个工具，请求用户重新考虑
  - 尝试其他方案
  - 告诉用户无法完成任务
```
