# MCP 基础介绍

## 什么是 MCP

MCP (Model Context Protocol) 是一个标准协议，让 AI 助手能够与外部工具和数据源安全地交互。

## 问题背景

想象你有一个很聪明的 AI 助手，但它被困在一个"盒子"里：
- 它看不到你的文件
- 它不能操作数据库
- 它不知道你公司的 API

**MCP 就是给这个 AI 装上的"手脚和眼睛"。**

## 传统方案的困境

```
传统方式：每个 AI 应用都要自己写插件
┌─────────┐
│ ChatGPT │ ──→ 写了一套文件插件
└─────────┘
┌─────────┐
│ Claude  │ ──→ 又写了一套文件插件（重复工作！）
└─────────┘

MCP 方式：统一标准，一次开发到处使用
┌─────────┐
│ ChatGPT │ ──┐
├─────────┤   ├──→ 同一个 MCP 服务器
│ Claude  │ ──┤
├─────────┤   │
│ 其他AI  │ ──┘
└─────────┘
```

## 核心架构

```
┌─────────────────────────────────────────────────────────────────┐
│                        整体架构                                   │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   用户 ───→ Agent ───→ 模型 ───→ MCP Server ───→ 外部资源      │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 各层职责

| 层级 | 职责 | 类比 |
|------|------|------|
| 用户 | 提出问题、审批操作、接收结果 | 老板 |
| Agent | 协调各组件、管理权限、控制流程 | 秘书 |
| 模型 | 理解问题、决策工具、生成回答 | 大脑 |
| MCP Server | 执行具体操作、返回结果 | 手脚 |
| 外部资源 | 文件、数据库、API 等 | 工作材料 |

## 核心价值

| 维度 | 传统方式 | MCP 方式 |
|------|----------|----------|
| 开发者 | 为每个 AI 平台写插件 | 一次开发，到处使用 |
| 用户 | 不同 AI 用不同工具 | 统一的工具生态 |
| 安全 | 插件权限难以控制 | 标准化权限审批 |
| 可维护性 | 分散的插件 | 集中的服务器 |
| 扩展性 | 各自为政 | 生态共享 |

## 关键特点

1. **标准化接口**：统一的 JSON-RPC 协议
2. **安全隔离**：服务器在独立进程中运行
3. **权限控制**：用户可审批每个工具调用
4. **可扩展**：任何人可以编写自己的 MCP 服务器
